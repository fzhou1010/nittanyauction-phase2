# Code Review Recommendations

> Brief overview of potential issues found after merging alex-branch, franks-branch, and elvis-branch into main. These are recommendations — not all are blocking.

---

## Bugs

### 1. Dead code / lost search feature — `routes/listings.py`
```python
return render_template('listings/browse.html', ...)

search = request.args.get("q")  # never runs
```
The old text search block (lines 42–60) sits after a `return` and is unreachable. The `?q=` search param is silently ignored. **Fix:** merge the text search logic into the new `browse()` before the return.

---

### 2. Schema syntax error — `schema.sql`
```sql
PRIMARY KEY (watchlist_id),product that you want it in
```
A comment got split across lines during a merge, leaving raw text in the SQL. SQLite will throw a parse error and fail to create the `Watchlist` table. **Fix:** remove `product that you want it in` — it belongs in the comment on the line above.

---

### 3. Broken HTML attribute — `templates/listings/browse.html`
```html
<input class="form-control me-3 required value="{{ request.args.get('q', '') }}">
```
The `value` attribute is trapped inside the `class` string. The search box will never be pre-filled and "required" becomes a literal class name. **Fix:** split into separate `class` and `value` attributes.

---

### 4. Wrong column key in category dropdown — `templates/listings/browse.html`
```html
<option value="{{ cat['Category'] }}">
```
The query now selects `category_name` from the `Categories` table, not `Category` from `Auction_Listings`. Every dropdown option will render blank. **Fix:** change to `cat['category_name']`.

---

### 5. Unreachable `except` block — `routes/auth.py` (local vendor registration)
```python
except Exception as e:
    ...
except sql.IntegrityError:   # never reached
    ...
```
`IntegrityError` is a subclass of `Exception`, so the broad catch always fires first. The specific handler is dead code. **Fix:** swap the order — `IntegrityError` first, `Exception` second.

---

## Refactoring Suggestions

### 6. Duplicate registration logic — `routes/auth.py`
The `bidder` and `student_seller` branches in `register_form` share nearly identical address/zipcode insert sequences. **Suggestion:** extract into a shared `_insert_address(db, ...)` helper to reduce duplication.

### 7. Password hashing defined in two places — `routes/auth.py` + `load_data.py`
`load_data.py` has a `hash_password()` function; `auth.py` reimplements the same logic inline. **Suggestion:** move `hash_password()` into `db.py` or a `utils.py` and import it from both.

### 8. Repeated auth guard — `routes/listings.py`
`if 'email' not in session: return redirect(url_for('auth.login'))` appears in every route. **Suggestion:** a `@login_required` decorator would eliminate the repetition.

### 9. Mismatched HTML tags — `templates/listings/listing.html`
`<h6>` closed with `</h3>`, `<h2>` closed with `</h5>`, etc. Harmless visually but invalid HTML. **Suggestion:** fix closing tags to match opening tags.

### 10. Stray IDE file committed — `tempCodeRunnerFile.py`
Auto-generated by VS Code's Code Runner extension. **Suggestion:** delete the file and add `tempCodeRunnerFile.py` to `.gitignore`.
